# 안전 리팩터링 (기능 변화 없음)

이 프로젝트는 **HTML · CSS · JavaScript만 사용하는 입문자용 테트리스**입니다.
**동작은 그대로 유지**하면서 코드 구조·가독성만 개선하세요.

## 목표

- 함수 분리·이름 정리·중복 제거·상수 정리
- 입문자가 읽기 쉬운 구조로 정돈
- **게임 규칙·점수·UI 동작은 변경하지 않음**

## 허용되는 변경

- 동일 로직을 더 명확한 함수로 추출 (예: `getMatrix`, `updateStats`)
- 매직 넘버를 기존 상수로 통합
- JSDoc·주석 보강 (한국어)
- dead code·미사용 변수 제거
- CSS 클래스 이름 정리 (HTML·JS 참조 함께 수정)
- `render()` 호출 경로 단순화 (동작 동일 조건)

## 금지되는 변경

- 점수 규칙·낙하 속도·충돌 규칙 변경
- 새 기능 추가 (다음 블록, 홀드, 사운드 등)
- 파일 분할·빌드 도구·외부 라이브러리 도입
- API/함수 시그니처를 깨는 대규모 재설계

## 작업 절차

1. 리팩터링 전 현재 동작을 짧게 요약합니다.
2. 변경 계획을 3~5개 항목으로 나열합니다.
3. **작은 단위**로 수정하고, 각 단계마다 동작 동일성을 설명합니다.
4. 리팩터링 후 수동 회귀 체크리스트를 제공합니다.

## 코딩 규칙

- 주석·커밋 메시지는 **한국어**
- JavaScript는 **ES Modules**만 사용
- 함수에는 **JSDoc** 유지 또는 추가
- 기존 네이밍·스타일과 맞출 것

## 출력 형식

```markdown
# 안전 리팩터링 결과

## 변경 요약
(기능 변화 없음을 명시)

## 변경 목록
| 파일 | 변경 내용 | 이유 |
|------|-----------|------|
| script.js | ... | ... |

## 동작 동일성 근거
- (어떤 입력·상태에서 결과가 같음을 설명)

## 회귀 확인 체크리스트
- [ ] ...
```
