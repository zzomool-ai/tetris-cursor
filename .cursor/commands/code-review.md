# 보드·블록 렌더링 코드 리뷰

이 프로젝트는 **HTML · CSS · JavaScript만 사용하는 입문자용 테트리스**입니다.
**보드와 블록 렌더링** 단계의 코드를 코드 리뷰 관점에서 점검하세요.

버그, 동작 회귀, 보안 이슈, 누락된 테스트를 우선합니다. 발견 사항이 응답의 중심이어야 하며, 심각도 순으로 정렬합니다.
**사용자가 명시적으로 요청하지 않는 한 코드를 수정하지 마세요.**

## 점검 범위

- `index.html` — 보드·UI DOM 구조
- `style.css` — CSS Grid 보드, 칸·블록 스타일
- `script.js` — `createPiece()`, `renderBoard()`, `drawPiece()`, `render()` 및 관련 헬퍼

## 점검 항목

### DOM 구조

- `#game-board`가 10열 × 20행 Grid 컨테이너로 올바르게 정의되었는지
- 칸(`cell`)이 JS에서 일관된 인덱스(row × COLS + col)로 접근되는지
- HTML id/class가 CSS·JS와 일치하는지
- 시맨틱 마크업·`aria-label` 등 기본 접근성이 갖춰졌는지

### 렌더링 함수

- `renderBoard()`가 `state.board`만 반영하고, 고정 블록과 떨어지는 블록의 책임이 분리되었는지
- `drawPiece()`가 보드 밖 좌표를 안전하게 건너뛰는지
- `render()` 호출 순서(`renderBoard` → `drawPiece`)가 올바른지
- `clearCell()` 후 클래스·`data-type`이 깨끗이 초기화되는지
- `buildBoardGrid()`가 중복 호출될 때 DOM이 깨지지 않는지

### 가독성·구조

- 상수(`COLS`, `ROWS`, `SHAPES`)와 상태(`state`)가 명확히 구분되는지
- 함수마다 JSDoc이 있고, 주석이 한국어인지
- 입문자가 따라 읽기 쉬운 함수 길이·이름인지
- ES Modules·외부 라이브러리 미사용 제약을 지키는지

### 버그·회귀 (렌더링 단계)

- 보드와 피스가 겹칠 때 화면이 깨지지 않는지
- 게임 시작/재시작 시 빈 보드와 새 블록이 올바르게 표시되는지
- 7종 블록(I, O, T, S, Z, J, L) 색상·모양이 모두 정의되어 있는지

## 출력 형식

한국어로 작성하세요.

```markdown
# 코드 리뷰: 보드·블록 렌더링

## 요약
(2~3문장)

## 발견 사항
| 심각도 | 위치 | 문제 | 권장 조치 |
|--------|------|------|-----------|
| High/Medium/Low | 파일:줄 또는 함수명 | ... | ... |

## 잘 된 점
- ...

## 테스트·검증 제안
- (자동 테스트가 없다면 수동 확인 시나리오)
```

## 제약

- **코드 변경 금지** (리뷰만 수행)
- 발견 사항이 없으면 "발견 사항 없음"을 명시하고, 잘 된 점을 구체적으로 기술하세요
- 아직 구현되지 않은 기능(낙하, 키보드 등)은 "미구현"으로 분류하고 버그로 보고하지 마세요
