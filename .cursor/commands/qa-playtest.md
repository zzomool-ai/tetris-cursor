# QA 플레이테스트 (라인 삭제·점수·게임오버)

이 프로젝트는 **HTML · CSS · JavaScript만 사용하는 입문자용 테트리스**입니다.
**라인 삭제, 점수, 레벨, 게임 오버, 재시작** 흐름을 사용자 시나리오 기준으로 점검하세요.

**사용자가 명시적으로 요청하지 않는 한 코드를 수정하지 마세요.**

## 점검 방법

1. `index.html`, `style.css`, `script.js`, `README.md`를 읽고 기대 동작을 파악합니다.
2. 각 시나리오에 대해 **기대 결과**와 **코드상 실제 동작**을 대조합니다.
3. 가능하면 `node --check script.js`로 문법을 확인합니다.
4. 브라우저 실행이 필요한 항목은 수동 검증 절차를 안내합니다.

## 테스트 시나리오

| ID | 시나리오 | 기대 결과 |
|----|----------|-----------|
| QA-01 | 페이지 로드 | 빈 보드(또는 시작 UI), 점수 0 |
| QA-02 | 게임 시작 | 현재 블록 표시, 게임 상태 `playing`(또는 동등) |
| QA-03 | 1줄 완성 | 해당 줄 삭제, 위 줄이 내려옴 |
| QA-04 | 2~4줄 동시 삭제 | 줄 수만큼 점수 가산(규칙에 맞게) |
| QA-05 | 점수 표시 | UI `#score`가 `state.score`와 일치 |
| QA-06 | 레벨 상승 | 줄 수 조건 충족 시 레벨·낙하 속도 변화 |
| QA-07 | 게임 오버 | 스폰 불가 시 오버레이/메시지, 입력 중단 |
| QA-08 | 다시 시작 | 보드·점수·레벨 초기화 후 재플레이 |
| QA-09 | 일시정지(P) | 낙하 정지, 재개 시 타이머 리셋 |
| QA-10 | 재시작 중 UI | 버튼·오버레이 상태가 README와 일치 |

시나리오는 프로젝트 구현 수준에 맞게 조정하되, **미구현 항목은 FAIL이 아니라 "미구현"**으로 표기하세요.

## 추가 점검

- README에 기술된 점수 규칙·조작법과 코드가 일치하는지
- 게임 오버 후에도 렌더링 루프·이벤트가 누수되지 않는지
- 연속 재시작 시 상태가 누적되지 않는지

## 출력 형식

한국어로 작성하세요.

```markdown
# QA 플레이테스트 보고서

## 요약
| 항목 | 결과 |
|------|------|
| PASS | N |
| FAIL | N |
| 미구현 | N |
| 판정 | 출시 가능 / 수정 필요 / 미완성 |

## 테스트 결과
| ID | 시나리오 | 기대 결과 | 결과 | 비고 |
|----|----------|-----------|------|------|
| QA-01 | ... | ... | PASS/FAIL/미구현 | ... |

## 실패·미구현 상세
(실패·주의 항목만 상세 설명)

## 수동 검증 안내
(브라우저에서 확인할 단계)
```

## 제약

- **코드 변경 금지**
- FAIL은 재현 가능한 근거(함수명·조건)를 함께 제시하세요
