# 게임 로직 리뷰 (낙하·충돌·조작)

이 프로젝트는 **HTML · CSS · JavaScript만 사용하는 입문자용 테트리스**입니다.
**낙하, 충돌 판정, 키보드 조작, 회전** 로직을 집중 점검하세요.

**사용자가 명시적으로 요청하지 않는 한 코드를 수정하지 마세요.**

## 점검 범위

- `script.js` — 게임 루프, 이동·회전·낙하, 충돌 검사, 이벤트 핸들러
- `index.html` — 조작법 안내와 실제 키 바인딩 일치 여부

## 점검 항목

### 충돌 판정

- `isValidPosition()`(또는 동등 함수)이 벽·바닥·고정 블록을 모두 검사하는지
- `y < 0`(스폰 영역)에서의 충돌 처리가 올바른지
- 착지(`lockPiece`) 시 보드 배열에 타입이 정확히 기록되는지
- 고정 직후 스폰 위치가 이미 막혀 있으면 게임 오버로 이어지는지

### 경계값

- `x = 0`, `x = COLS - 1`에서 좌우 이동·회전
- `y = 0`(최상단)에서 회전·이동
- I·O 등 크기가 다른 블록의 스폰 X 좌표
- 보드 밖 음수 좌표·`COLS`/`ROWS` 초과 좌표 접근 방지
- 게임 시작 직후·일시정지 해제 직후 낙하 타이밍(`lastDrop` 등)

### 회전

- 4방향 회전이 모든 블록 타입에서 동작하는지
- 회전 행렬/상태 인덱스가 `SHAPES` 정의와 일치하는지
- 벽 차기(wall kick) 또는 대안 로직이 좁은 공간에서 오동작하지 않는지
- O 블록 회전 시 위치가 어긋나지 않는지

### 낙하·조작

- 자동 낙하 간격이 레벨/상태에 맞게 적용되는지
- 소프트 드롭(↓), 하드 드롭(Space)이 충돌 후 올바르게 멈추는지
- `menu`/`gameover`/`paused` 상태에서 입력이 무시되는지
- `requestAnimationFrame` 루프가 중복 등록되지 않는지
