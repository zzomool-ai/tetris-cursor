# 버그 헌트 (고위험 버그 제거)

이 프로젝트는 **HTML · CSS · JavaScript만 사용하는 입문자용 테트리스**입니다.
게임 플레이·상태·렌더링에서 **고위험 버그**를 찾아 수정하세요.

## 목표

- 크래시, 무한 루프, 상태 꼬임, 점수 오류, 화면 깨짐 등 **플레이를 망가뜨리는 버그** 제거
- 수정 범위는 **버그 수정에 필요한 최소 변경**으로 제한

## 점검·수정 우선순위

### High (즉시 수정)

- 게임 중 JS 예외로 멈춤
- 충돌 판정 오류로 벽·블록 관통
- 게임 오버/재시작 후 상태가 초기화되지 않음
- `requestAnimationFrame`·`setInterval` 중복으로 낙하가 배속됨
- 보드 배열 인덱스 범위 초과

### Medium

- 일시정지 중 입력·낙하가 계속됨
- 줄 삭제 후 보드·점수 불일치
- 회전·벽 차기 오동작
- 렌더링 순서 문제로 블록이 사라지거나 겹쳐 보임

### Low (명확한 버그만)

- UI 텍스트와 실제 점수 불일치
- 버튼·오버레이 표시 상태 오류

## 작업 절차

1. `script.js` 중심으로 버그 후보를 식별합니다.
2. 재현 조건과 원인을 짧게 기록합니다.
3. **최소 diff**로 수정합니다.
4. 수정 후 영향 받는 시나리오를 나열합니다.

## 코딩 규칙

- 주석·커밋 메시지는 **한국어**
- JavaScript는 **ES Modules**만 사용
- 수정·추가 함수에 **JSDoc** 작성
- **외부 라이브러리 추가 금지** (필요 시 사용자 동의 요청)

## 출력 형식

```markdown
# 버그 헌트 결과

## 수정한 버그
| 심각도 | 문제 | 원인 | 수정 요약 |
|--------|------|------|-----------|
| High/Medium/Low | ... | ... | ... |

## 수정하지 않은 항목
- (의도적 미구현·리팩터링 성격은 여기에)

## 회귀 확인
- [ ] 게임 시작/재시작
- [ ] 이동·회전·낙하
- [ ] 줄 삭제·점수
- [ ] 게임 오버
```

## 제약

- **기능 추가·대규모 리팩터링은 하지 마세요** (구조 개선은 `/refactor-safe` 사용)
- 불확실한 버그는 추측 수정 대신 재현 단계를 보고하세요
- 테스트 파일이 없으면 수동 확인 체크리스트를 제공하세요
